## Тестирование веб-приложения Яндекс.Маршруты  
Яндекс.Маршруты - сервис построения маршрутов, расчёта времени и стоимости поездки для разных видов транспорта.

**Задача:** функциональное и кроссбраузерное тестирование фичи «Каршеринг»

**Выполненные задачи:**
1. Проведен анализ и декомпозиция требований и макетов, уточнены серые зоны.
2. Разработана тестовая документация:
   - чек-лист на вёрстку формы бронирования и элементов на карте (иконки авто и действия с ними)
   - чек-лист на логику окон «Способ оплаты» и «Добавление карты»
   - тест-кейсы на кнопку «Забронировать»
3. Проведено тестирование.
4. Заведены баг-репорты.

**Инструменты** 
•	DevTools
•	Figma 
•	Google Таблицы 
•	Стенд-тренажёр Яндекс.Практикум

## Тестирование мобильного приложения Яндекс.Метро
Яндекс.Метро — сервис, который позволяет ориентироваться в метро с помощью мобильного устройства. В приложении есть схема метро, которая помогает построить маршрут и оценить время в пути; в приложении появляются актуальные уведомления о работе станций метро и изменениях графика работы.

**Задача:** функциональное и регрессионное тестирование мобильного приложения после рефакторинга кода

**Выполненные задачи:**
1. Проведен анализ и декомпозиция требований и макетов к функциональности, затронутой рефакторингом, уточнены серые зоны.
2. Разработана тестовая документация:
   - чек-лист к функциональности, затронутой рефакторингом
   - чек-лист регрессионного тестирования, учитывающий особенности мобильного приложения
3. Проведено тестирование в эмуляторе.
4. Заведены баг-репорты и составлен отчет о тестировании.

**Инструменты**

## <a name="diploma"></a> Диплом - Комплексное тестирование сервиса Яндекс.Самокат (web, mobile app, REST API, DB)
Яндекс.Самокат — сервис, который позволяет арендовать самокат на несколько дней.

**Задачи:** 
- функциональное, исследовательское и кроссбраузерное тестирование web-интерфейса
- функциональное тестирование mobile-приложения на Android и специфичные проверки взаимодействия с устройством — прерывания, фоновый и офлайн-режимы
- функциональное тестирование REST API

**Выполненные задачи по клиентскому веб-приложению:**
1. Проведен анализ и требований и макетов, уточнены серые зоны, составлен mainmap для декомпозиции экрана «Сделать заказ»
2. Разработана тестовая документация:
   - спроектированы проверки (применены техники КЭ и ГЗ тест-дизайна) для валидации полей экрана «Сделать заказ»
   - составлен чек-лист к функциональности экрана «Статус заказа»
3. Проведено тестирование всего веб-интерфейса по макетам и требованиям (кроме лэндинга) Использован Charles Proxy для мокирования ответов логики смены статусов доставки до готовности бэкенда
4. Дефекты локализованы (использованы DevTools, Postman, SQL-запросы к БД через CLI), оформлены баг-репорты

**Выполненные задачи по мобильному приложению для курьеров:**
1. Проведен анализ и декомпозиция требований и макетов, уточнены серые зоны
2. Разработана тестовая документация:
   - тест-кейсы для функциональности и верстки нотификаций и отсутствия интернет-соединения, особенностей мобильного приложения
3. Проведено тестирование на эмуляторе в Android Studio 
4. Дефекты локализованы (использованы Postman, SQL-запросы к БД через CLI), оформлены баг-репорты.

**Выполненные задачи по API:**
1. Проведен анализ требований к новой функциональности бэкенда, изучена документация к API в apiDoc
2. Разработана тестовая документация:
   - чек-лист для ручек создания и удаления курьера, получения заказа по его номеру
3. Проведено тестирование API через Postman, выполнены SQL-запросы к базе данных (PostgreSQL) через командную строку (CLI) для проверки корректности данных
4. Оформлены баг-репорты
